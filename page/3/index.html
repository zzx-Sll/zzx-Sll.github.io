函数的声明式

发表于 2021-12-13 分类于 前端

函数的声明方式在js中只有三种

  • 1 通过函数构造器声明
  • 2 函数表达式来声明
  • 3 函数声明
  • 注意：
    • 其中，可以进行变量解析提升的只有函数声明方式。通过函数表达式来声明的函数无法进行函数提升。
    • 不建议在块级作用域中定义函数，如果非要定义函数只能通过函数表达式来定义函数
    • 函数表达来定义函数funciton 后不能定义名字。如 var f=function g(){},此时通过g来访问函数时无效的。typeof打印g为undefined。打印g()会报错。
    • 示例：
      •     var f = function g() {
        return 23;
     };
    typeof g(); // Error

requestAnimationFrame之渲染大量数据

发表于 2021-12-01 分类于 前端

requestAniamationFrame之渲染大量数据

  • 优点

    • CPU 节能：使用 setTimeout 实现的动画，当页面被隐藏或最小化时，setTimeout 仍然在后台执行动画任务，由于此时页面处于不可见或不可用状态，刷新动画是没有意义的，完全是浪费 CPU 资源。而 requestAnimationFrame 则完全不同，当页面处理未激活的状态下，该页面的屏幕刷新任务也会被系统暂停，因此跟着系统步伐走的 requestAnimationFrame 也会停止渲染，当页面被激活时，动画就从上次停留的地方继续执行，有效节省了 CPU 开销

    • 函数节流：在高频率事件(resize,scroll 等)中，为了防止在一个刷新间隔内发生多次函数执行，使用 requestAnimationFrame 可保证每个刷新间隔内，函数只被执行一次，这样既能保证流畅性，也能更好的节省函数执行的开销。一个刷新间隔内函数执行多次时没有意义的，因为显示器每 16.7ms 刷新一次，多次绘制并不会在屏幕上体现出来。

typeof注意事项

  •         // typeof 运算符返回一个操作后的字符串
        var test = 1; typeof test == 'number'; // true
        var test = 1.2; typeof test == 'float'; //false
        var test = undefined; typeof test == 'undefined'; //true

        var test = null; typeof test == 'object'; //true
        /*
        typeof 输出的结果
         Undefined => undefined
         Null      => object //从javascript创立便如此。null为一个空指针。所以是对象结构

         BigInt    => bigint

有关判断的问题

发表于 2021-11-11 更新于 2022-02-10 分类于 前端

有关判等的问题

  •           console.log(([]) ? true : false);
          console.log(([] == false ? true : false));
          console.log(({ a: 2 } == false) ? true : false);
          /*
          output: true true false
          首先了解一下：Boolean()判断为false的情况
           null、undefined、NaN、0、''。其他情况下判断都为true
          其次 == 在进行判断时会进行类型转换。而它一般喜欢转为数字
          第一题:  Boolean([]) 为true
          第二题: false 转为0, 而[]在进行数组转换前先转为'' 空字符串转为0 所以第二题是 0==0?true:false=》true
          第三题: false转为0 而空对象转为 NaN，所以NaN==0?true:false =>false
          */
  • ‘>’比较运算符是转为ASCII码来进行比较的
  • ==比较时，喜欢转为数字来进行比较。

  • ```javascript

                       /* 
            函数的作用域：只有函数有作用域，函数的作用域在函数声明写好时就已经确定了！
            js的预解析: js并不是一味的从上到下执行。在执行之前会有个预解析过程。预解析就是变量、函数的声明会提升到当前作用域的顶部的过程
               注意：!!!! 
                1:当函数同名时，后提升的函数会覆盖前一个函数！！！
                2：函数和变量重名时，只会提升函数，忽略变量 ！！！
                3：函数表达式不会被提升！！！
                4：预解析是分作用域的！！！
                5： 条件式函数申明在不同浏览器中表现不同所以不要在条件表达中声明函数！！！
            */

            // 函数作用域的示例
            var num = 123;
            function foo1() {
              console.log(num);
            }

            function foo2() {
              // console.log(num);
              var num = 456;
              foo1() // num=123
            }
            foo2()

            /* 
             以下示例可正常执行。就是因为变量提升,函数的声明会提升到顶部。之后才是函数的调用
            */
            foo()
            function foo() {
              console.log('我可以执行');
            }
            /* 
             函数表达式不会被提升！！！
            */
            (

                var x = foo();
                // 函数表达式不会提升！！！
                var foo = function foo() {
                  console.log('foo函数被执行了');
                  return 'footbar'
                }
                foo()
                console.log({ x });
                return x
              }
            )()

函数参数作用域和函数作用域的关系

  • 首先我们先看代码
    •     function m(a = 'aaa', n = function () 
     {
          console.log(a); // 'aaa'
          a = 'bbb'
          console.log(a);  // 'bbb'
         }) {
           var a = 'ccc'
           console.log(a); //'ccc'
           n()
           console.log(a);  // ccc
      }
      m()
      /* 
       ccc
       aaa
       bbb
       ccc
      */
      • 产生这种情况的原因是因为ES6中参数的默认值会产生一个参数作用域，并且该作用域与函数作用域是独立的。所以在函数体内是访问不到参数体内的变量
      • 但是，当我们在函数体内用let m=’ccc’时。会发现浏览器出现报错。为什么明明是两个独立作用域却还是会报错呢？
      • 后面在网上查看到了一个解释比较有信服力。下面让我们看下
      • 我们不希望这个函数只是因为给a参数加了默认值就报语法错误，所以就只好选择允许新作用域上产生一个重名变量。虽然var a和a参数实际是两个变量，我们又要保持原有的行为（即在var a声明之前访问a的结果就是a参数的值），这意味着var a的初始值必须就是a参数的初始值。最终就导致了题主的代码片段2这样看似诡异的结果。好在ES6的实践上我们已经不再使用var，而是全部转向let/const，所以在工程上避免了这种很让人confuse的情况。

webpack打包的原理

手写一个简单的webapck

  • 准备工作：
    • 新建一个项目名文件夹
    • npm init -y 初始化一个项目
    • 根目录下建立一个 bin 文件夹
    • 在bin目录下建立一个打包的主程序js文件（文件名自定义）
    • 在主程序文件下顶部对该程序文件进行声明当前的运行环境为node #!/usr/bin/env node
    • 在package.json文件下进行bin节点的声明，建立自己的打包脚本指令，指定打包的文件路径。
    • npm link 将自己打包程序文件链接到全局
    • 然后执行打包指令就可以在node环境下输出了
