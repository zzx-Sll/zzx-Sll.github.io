枚举类型

  • 枚举类型是ts中新增的一种数据类型。它可以认为是用来表示一个简单对象所有取值可能的集合。
  • 枚举类型用关键词enum来指定
    • <!-- 日常生活的星期 -->
enum days{sun,mon,tues,web,thur,fri,sat}
days.sun===0 //true
days[0]===sun //true
  • enum默认位置是从0开始的，也可以手动指定。
  • enum指定可以为数字也可以为字符串可以为异构（数字和字符串，从字符串开始全为字符串）
  • 枚举类型的使用场景
    • 用于后端返回数字对应值的情况，如0、1代表true、false。这时我们可以用enum来存储，以便于后期维护

使用XLSX导出el-table表格数据重复的问题

  • 问题描述：xlsx导出el-table表格数据时，出现数据重复导出问题，经查验，table表格数据无重复，导出方式也无问题。最后经查阅发现，如果给el-table表格中的列添加了fixed属性，在导出时会导致表格重复。

  • 解决方法：在导出前做校验，移除fixed属性，然后再导出就不会出现数据 重复了。

  • 代码：

      table.removeChild(table.querySelector(".el-table__fixed"));
      let vb = XLSX.utils.table_to_book(table, { raw: false });

      let vbout = XLSX.write(vb, {
        bookType: "xlsx",
        bookSST: false,
        type: "array"
      });

      try {
        FileSaver.saveAs(
          new Blob([vbout], { type: "application/octet-stream" }),
          this.$root.stationType.NM +
            " 年特征值表(" +
            (this.date.length > 4
              ? this.formatDate(new Date(this.date), "yyyy年MM月")
              : this.formatDate(new Date(this.date), "yyyy年")) +
            ").xlsx"
        );
      } catch (e) {
        if (typeof console !== "undefined") console.log(e, vbout);
      }
      return vbout;

获取元素距离可视区域的距离：element.getBoundingClientRect()

  • 该方法返回一个rect对象，该对象是由该元素的getClientRect()方法返回的一组矩形集合，包含完整元素的最小矩形。rect是一个具有left、right、top、bottom的DOMRect对象。
  • left、right、top、bottom表示当前盒子距离可视区域的各个方向距离
  • 当计算边界矩形时，会考虑到当前视口区域的滚动操作，也就是说当滚动位置发生了改变，rect的对象值也会随之发生改变。如果你需要获得相对于当前页面左上角的的坐标值，只需给left、top加上window.scrollLeft、window.scrollTop（为了照顾浏览器的兼容性可以使用window.pageXoffset和window.pageYoffset替代）。就可以得到与当前滚动位置无关的值。
  • 获取可视区域的相关数据:window.innerHeight、window.innerWidth、document.documentElement.clientHeight、document.documentElement.clientWidth
  • 根据这些信息可以实现一些如让元素滚动到可视区域再加载动画或者图片的操作

setState()说明

更新数据
  • setState()更新数据是异步的(相对来说是异步的)
  • 注意：使用该语法，后面的setState不要依赖前面的setState的值
  • 多次调用setState，只会触发一次render
setState是异步的么？
  • setState不是异步的，它没有用到异步的模块。但是它是批量更新的。
  • setState在批量更新函数，在同一个React的生命周期钩子函数中使用时是相对异步的。上一次的setState的值，下一次不能立马拿到。
  • setState在异步模块中是相对同步的，上一次的setState的值，下一次可以立马拿到。
推荐语法
  • 使用setState((state,props)=>{})语法
  • 参数 state:表示最新的state
  • 参数props:表示最新的props
第二个参数
  • 场景：在状态更新后（页面完成渲染后立即执行某个操作）与 componentDidUpdate钩子做的功能有时可以相互替代
jsx语法的转换过程
  • jsx仅仅是createElement()方法的语法糖
  • jsx语法被@babel/preset-react插件编译为createElement（）方法
  • React元素：是一个对象，用来描述你希望在屏幕上看到的内容
组件更新机制
  • setState()的两个作用
  • 修改state
  • 更新组件
  • 过程父组件更新自己时，同时会更新其所有子组件，但对兄弟组件无影响
组件性能优化
减轻state
  • 减轻state:只存储渲染相关的数据
  • 注意：不用做渲染的数据不要放在state中
  • 对于这种需要多个方法中用到的数据，用this.xxx存储起来
避免不必要的重新渲染
  • 组件更新机制：父组件更新会引起子组件也被更新，这种思路很清晰
  • 问题：子组件没有任何变化时也会更新
  • 如何避免不必要的渲染？
  • 解决方式：使用钩子函数 shouldComponentUpdate(nextProps,nextState)
    • 在这个函数中，nextProps和nextState是最新的状态及属性
  • 作用：这个函数有返回值，如果返回true，则代表需要重新渲染，如果返回false，代表不需要重新渲染
  • 触发时机：更新阶段的钩子函数，组件重新渲染前执行(比render早)
    纯组件
纯组件以及使用
  • 纯组件：PureComponent 与 React.Component功能相似
  • 区别：PureComponent 内部自动实现了shouldComponentUpdate钩子。不需要手动比较
  • 原理：纯组件内部自动对比了前后两次的props,state的值，来决定要不要更新组件
实现原理
  • 说明：纯组件内部的对比是shallow compare（浅层对比）
  • 对于值类型来说：比较两个值是否相同
  • 引用类型：只对比对象的引用地址是否相同
  • 注意：state中或props中属性值为引用类型时，应该创建新数据，不要直接修改原数据

React组件的生命周期

组件的生命周期：组件从创建到挂载到卸载的过程

创建时的生命周期

  • constructor:构造函数
    • 作用：设置state数据，this的指向设置
  • render：渲染函数
    • 作用：渲染页面。注意：不要在此函数中使用setState()
  • componentDidMount:组件挂载到页面后触发的函数
    • 作用：1 发送网络请求 2：获取dom结构
  • 执行的顺序：constructor–>render–>componentDidMount

组件更新时的生命周期

  • render():每次组件渲染都会触发
    • 作用：渲染ui
  • componentDidUpdate():组件更新后触发
    • 作用：1发送网络请求 2DOM操作 3：如果使用setState()必须放在一个if条件中，发送网络请求同理
    • 注意：if条件可以是判断props的不同，上次的props：prevProps,此次的props：this.props

组件卸载时的生命周期

  • componentWillUnmount:组件卸载后触发
    • 作用：执行清理工作（比如：清理定时器等）

React抽离事件处理程序时解决this的指向问题

  • 抽离事件处理程序时this指向undefined的问题
  • 原因：
  1. jsx中this没有默认绑定，需要使用bind绑定，否则this会是undefined。

由于类的方法默认不会绑定this，因此在调用的时候如果忘记绑定，this的值将会是undefined。

通常如果不是直接调用，应该为方法绑定this。

  1. 这个问题的本质原因如下：

这不是React的原因，这是JavaScript中本来就有的。如果你传递一个函数名给一个变量，然后通过在变量后加括号()来调用这个方法，此时方法内部的this的指向就会丢失。
3. 在React（或者说JSX）中，传递的事件参数不是一个字符串，而是一个实实在在的函数：

这样说，React中的事件名（eg:onClick、onChange）就是所举例子中的中间变量，React在事件发生时调用onClick，由于onClick只是中间变量，所以处理函数中的this指向会丢失，其实真正调用时并不是this.handleClick(),如果是这样调用那么this指向就不会有问题。真正调用时是onClick()。

gojs中的图层级别(layer)

标准层

每个图都从几个标准层开始，这些都是它们的Layer.name,按从最后面到最前面的顺序排列：

  • “Grid”:保存了Diagram.grid和您希望位于所有内容之后的任何其他静态零件(“网格”图层中的所有零件都是自动不可选择的，因为该图层的Layer.allowSelect为false 。）)
  • “BackGround”:背景层
  • “ForeGround”:前景层
  • “Adornment”,保持装饰物的选择和各种工具
  • “Tool”:持有用于执行各种工具的零件

每个零件根据其Part.layerName放置字一个Layer中，默认值为空字符串。使用 Diagram.findLayer查找到具有图层名称的图层。通过设置Part.layerName更改零件所在的层。

由于layer.isTemporary为true，UndoManager会自动忽略对“Grid”，“Adornment”,和”Tool”层中的零件所做的更改。

由于Layer.allowSelect为false，因此无法选择”Grid”图层中的零件。这样可以放置用户选择可见的背景网格。

如果你想在选择节点时改变他的层级，由于每个Part都有一个Part.selectionChanged函数，您可以在这个函数中设零件选中时改变他的层级，未选中时恢复层级

同时，layer层级改变时也有相应的函数可以在层级改变后做一些事情。可以在layerChanged的函数中进行操作。
