前端自动化导入文件

发表于 2021-12-25 分类于 前端 
  •  /* 
    import default from './../App';
 前端自动化导入:当项目比较大的时候我们需要拆分为多个功能模块。一个一个导入的话，如果文件特别多就会比较耗时间和造成代码过多，如果我们使用的打包工具是webpack的话我们可以利用webpack的自动导入函数 require.context来实现这些模块的自动导入
 require.context(directory,useSubdirectories,regExp)函数有三个参数:
   directory:需要导入的路径。必须为字符串
   useSubdirectories:使用查找下面子文件 值为Boolean
   regExp:匹配文件的正则表达式
  该方法返回一个函数
 */

 /**
  * 定义解析自动化导入的函数
  * @param {*} fn :传入webpack自动化导入解析文件后的函数
  * @returns  使得 ./moduleA.js=> moduleA 返回经过解析后的以moduleA:fn(key).defalut类型的包含当前文件路径下的所有子文件对象.
 */
  function directoryParse(fn) {
   return fn.keys().reduce((modules, modulePath) => {
  // 正则替换： ./moduleA.js||.c/moduleA.js=>moduleA
  let moduleName = modulePath.replace(/(.*\/)*([^.]+).*/gi, '$2');
  modules[moduleName] = fn(modulePath).default;
  return modules;
  }, {});
  }

 export default directoryParse;

  • 在需要引入解析后文件中导入函数并且执行函数就行了

v-model绑定计算属性?

发表于 2021-12-22 分类于 前端

  • 众所周知，计算属性是不能被修改的。只能对齐求值。其原因是在对计算属性进行数据劫持时对setter进行了劫持报错。而v-model是vue中双向绑定数据的语法糖。所以当v-model绑定了一个计算属性时会报错。那么，难道真的不能将计算属性绑定到v-model上去么？

  • 答案是非也。其实vue官方文档也考虑到这种情况了，所以还是给我们提供了办法的。computed属性并不一定就是一个函数，可以是一个对象(vue3中同理),而vue给我们提供了get、set两种方法。其实我们平时直接定义函数的方式就是默认使用了get函数。所以我们可以使用对象来分别使用get进行求值，用set来进行修改值。但是明明vue中对setter进行了劫持，不能修改计算属性的值，为什么还提供方法给我们修改值呢？其实这个set方法并非直接对计算属性的值进行了修改。而是我们使用计算属性时是根据依赖的数据来进行操作求值返回的，而在set方法中我们可以根据传入的新值反过来推断此时依赖的值。通过修改依赖值进而达到修改计算属性值的目的。顺便提一句,计算属性也并非不能传参，我们可以利用return 一个函数，而这个函数可以接收参数。变相达到接收参数的目的。但是计算属性绑定到v-model上时不能传参(不知道是不是我的方法不对?)

  • 题目:

    •    function Foo() {
     getName = function () { console.log(1) }
     return this
     }
     Foo.getName = function () { console.log(2) }
     Foo.prototype.getName = function () { console.log(3) }
     var getName = function () { console.log(4) }
     function getName() { console.log(5) }

     //请写出以下输出结果：
     Foo.getName()
     getName()
     Foo().getName()
     getName()
     new Foo.getName()
     new Foo().getName()
     new new Foo().getName()

     /* 
      答案: 2、4、1、1、2、3、3
      首先 var getName 会进行提升 但是会被 function getName提升覆盖。
      1 Foo.getName()指向 此时会找Foo上的getName函数此时 输出 2
      2 getName()虽然之前提升了函数但是后面 var getName=function(){}执行，对 getName重新进行了赋值，覆盖了之前的函数，此时输出 4 
      3 Foo().getName()首先执行 Foo()，函数体执行 由于此时getName()未显示定义所以getName()是全局变量，又会覆盖之前的getName() 而return 的this指向window，所以执行getName() 输出 1
      4 再次执行 getName() 全局的getName()未发生改变 输出 1 
      5 执行 new Foo.getName() 由于new 和 .运算符优先级一样，从左到右执行，但是new是用来调用函数的s所以 执行的是new (Foo.getName()) 输出的是 2
      6 由于new 和 .运算符优先级一样，从左到右执行。执行的是 (new Foo()).getName 生成了 Foo实例,然后执行 getName方法，而实例会通过原型链查找到 Foo.prototype上的getName方法输出 3
      7 同步骤6一样，通过原型链查找输出 3
      */

  • 解析:
    • 知识点前提: 赋值操作顺序是从右到左,b=a是浅拷贝。a,b指向的是堆上的同一个地址(同一个对象)
      • ```javascript
        var a=1,b=2,c=3;
        a=b=c
        console.log(a,b,c) // ouput 3,3,3
    • 执行 a.x操作
      • ```javascript
        a.x //此时a 为旧对象 {n:1}。注意:此时a被解析为旧对象！！！
        // 所以这步执行后结果为 a={n:1,x:undefined} b={n:1,x:undefined}
    • 执行 a.x=a
    • 最后执行 console.log(a.x) console.log(b.x),此时又会重新解析a,b 而a={n:2},访问对象上不存在的属性输出undefined,而此时b的值为{n:1,x:{n:2}},所以访问b.x 输出 {n:2}
