有关于原型对象的问题

发表于 2022-02-21 分类于 前端

问题描述：

  • 最近在工作之余写代码发现了一个问题，当一个对象被创建或者实例化后。去更改他的原型对象，但是测试发现后他仍然指向未发生改变之前的原型对象。
  • 按照常理，protoptye是一个对象，当我们覆盖了原来的对象后，实例的原型应该指向被修改后的原型对象才对。但是经过测试发现并不是。经过查阅资料后才发现，Function用于new构造对象时的处理过程，对象的[[prototype]]在构造时已经设定完毕，而不是每次访问都去查找构造函数。而在修改之后创建的对象访问的则是被修改之后的原型。

NaN的判断问题

  • NaN==NaN (false)
  • 数组的indexOf方法不能查找NaN
  • 数组的includes方法认为NaN等于自身，所以可以查找NaN
  • set 数据结构认为NaN等于自身
  • 使用Object.is可以判断NaN等于NaN即 Object.is(NaN,NaN) （true）
  • Object.is还可以区分+0,-0. Object.is(+0,-0) (false)

let、const声明的全局变量不是顶级对象的属性

发表于 2022-02-09 分类于 前端

es5中使用var声明的全局变量为顶级对象window的属性
es6中使用let、const声明的全局变量不再是window的属性了

  • ```javascript
    var a = ‘w’
    let obj = {
    a: ‘o’,
    print: function () {
    console.log(this);
console.log(this.a);
    },
    }
    let p = obj.print;
    obj.print(); // o
    p(); // w
  • 上面两段代码输出的结果不同的原因就是因为，p()调用时this都是指向window。区别是var声明的全局变量是window对象的一个属性。而使用let定义的全局变量不是window对象的属性。所以一个输出w一个输出undefined

this指向面试题

发表于 2022-02-08 分类于 前端

this的指向问题：函数的this指向
抛开es6新增的genrator函数，函数分为两种，普通函数和箭头函数

  • 普通函数调用时this指向：

    • 做为函数调用（函数自身调用）函数的上下文this指向全局对象window。严格模式下指向undefined
    • 做为属性调用 xxx.func() 函数上下文指向函数的调用者
    • 作为构造函数调用 this指向 new 出来的实例对象

  • 箭头函数调用时this指向：

    • 由于箭头函数没有自己的作用域，所以箭头函数的this指向定义箭头函数时最近一级的函数上下文。可以通过下面例子来理解

    •       function Test() {
     this.flag = false
     this.change = () => {
       console.log('this=>', this); // btn实例对象
       this.flag = true
     }
   }
   let btn = new Test()
   btn.change()
   console.log(btn.flag); // true 此时 change是箭头函数，this指向定义箭头函数的函数最近一级函数上下文Test。但是由于Test为构造函数，函数上下文指向实例对象，所以this仍然是指向btn的
   console.log(btn);

   console.log('---------------------');
   function test2() {
     return {
       flag: false,
       change: () => {
         console.log('this=>', this); //window
         this.flag = true
       }
     }
   }
   let obj = test2();
   obj.change()
   console.log('flag', obj.flag); // false 此时change为箭头函数，this指向test的函数上下文 而test自调用，this指向全局对象window. 所以 flag输出为 false
   console.log({ obj });
