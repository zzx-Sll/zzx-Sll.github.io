          output: true true false
          首先了解一下：Boolean()判断为false的情况
           null、undefined、NaN、0、''。其他情况下判断都为true
          其次 == 在进行判断时会进行类型转换。而它一般喜欢转为数字
          第一题:  Boolean([]) 为true

                函数的作用域：只有函数有作用域，函数的作用域在函数声明写好时就已经确定了！
                js的预解析: js并不是一味的从上到下执行。在执行之前会有个预解析过程。预解析就是变量、函数的声明会提升到当前作用域的顶部的过程
                   注意：!!!! 
                    1:当函数同名时，后提升的函数会覆盖前一个函数！！！

                    3：函数表达式不会被提升！！！
                    4：预解析是分作用域的！！！
                    5： 条件式函数申明在不同浏览器中表现不同所以不要在条件表达中声明函数！！！
                */

函数参数作用域和函数作用域的关系

  • 首先我们先看代码
    •               function m(a = 'aaa', n = function () {
       console.log(a); // 'aaa'
       a = 'bbb'
       console.log(a);  // 'bbb'
      }) {
        var a = 'ccc'
        console.log(a); //'ccc'
        n()
        console.log(a);  // ccc
      }
      m()
      /* 
       ccc
       aaa
       bbb
       ccc
      */
      • 产生这种情况的原因是因为ES6中参数的默认值会产生一个参数作用域，并且该作用域与函数作用域是独立的。所以在函数体内是访问不到参数体内的变量
      • 但是，当我们在函数体内用let m=’ccc’时。会发现浏览器出现报错。为什么明明是两个独立作用域却还是会报错呢？
      • 后面在网上查看到了一个解释比较有信服力。下面让我们看下
      • 我们不希望这个函数只是因为给a参数加了默认值就报语法错误，所以就只好选择允许新作用域上产生一个重名变量。虽然var a和a参数实际是两个变量，我们又要保持原有的行为（即在var a声明之前访问a的结果就是a参数的值），这意味着var a的初始值必须就是a参数的初始值。最终就导致了题主的代码片段2这样看似诡异的结果。好在ES6的实践上我们已经不再使用var，而是全部转向let/const，所以在工程上避免了这种很让人confuse的情况。

webpack打包的原理

手写一个简单的webapck

  • 准备工作：
    • 新建一个项目名文件夹
    • npm init -y 初始化一个项目
    • 根目录下建立一个 bin 文件夹
    • 在bin目录下建立一个打包的主程序js文件（文件名自定义）
    • 在主程序文件下顶部对该程序文件进行声明当前的运行环境为node #!/usr/bin/env node
    • 在package.json文件下进行bin节点的声明，建立自己的打包脚本指令，指定打包的文件路径。
    • npm link 将自己打包程序文件链接到全局
    • 然后执行打包指令就可以在node环境下输出了

清除定时器误差

发表于 2021-11-04 分类于 前端

清除定时器误差

  • 问题：由于定时器是异步宏任务，所以并不是真的隔了设置的时间间隔就触发，而是会等待所有同步任务执行完后才会执行。故存在一定的误差

  • 解决方法：

    • 向服务器轮询查询误差。但是每秒都轮询会造成性能损耗。如果访问量过大会造成服务器压力

    • 前端手动解决。代码：

      •              /* 前端根据时间差来自动调整间隔时间的方式来清除定时器误差
              1 设置变量记录倒计时次数
              2 记录起始的时间
              3 记录定时器开始调用的时间
              4 设置interval变量为1000ms
              5 利用定时器开始的时间减去起始时间*倒计时次数即得到倒计时误差
              6 设置下一次执行定时器的间隔时间为 interval-上一次定时器误差*/
             //  初始化时间间隔
             let Interval = 1000;
             /* 记录执行次数的变量 */
             let count = 0;
             /* 开始倒计时的时间 */
             let startTime = new Date().getTime()
             // 计时器名称
             let timer;
             /* 距离服务器活动开始的时间 */
             let ms = 5000;
             /* 误差 */
             let offset;
             if (ms > 0) {
               timer = setTimeout(clearOffset, Interval)
             }
             function clearOffset() {
               count++;
               offset = new Date().getTime() - (startTime + count * Interval)
               console.log('当前时间', new Date().getTime());
               console.log('起始时间', startTime);
               let nextInterval = Interval - offset
               if (nextInterval < 0) {
                 nextInterval = 0
               }
               ms -= Interval
               console.log(`'执行次数${count}',误差${offset},下一次执行间隔时间${nextInterval},倒计时还剩${ms}`);
               // 判断，倒计时时间是否到达，如果到达则清除定时器
               if (ms < 0) {
                 clearTimeout(timer)
               } else {
                 timer = setTimeout(clearOffset, nextInterval)
               }

             }
