React抽离事件处理程序时解决this的指向问题

  • 抽离事件处理程序时this指向undefined的问题
  • 原因：
  1. jsx中this没有默认绑定，需要使用bind绑定，否则this会是undefined。

由于类的方法默认不会绑定this，因此在调用的时候如果忘记绑定，this的值将会是undefined。

通常如果不是直接调用，应该为方法绑定this。

  1. 这个问题的本质原因如下：

这不是React的原因，这是JavaScript中本来就有的。如果你传递一个函数名给一个变量，然后通过在变量后加括号()来调用这个方法，此时方法内部的this的指向就会丢失。
3. 在React（或者说JSX）中，传递的事件参数不是一个字符串，而是一个实实在在的函数：

这样说，React中的事件名（eg:onClick、onChange）就是所举例子中的中间变量，React在事件发生时调用onClick，由于onClick只是中间变量，所以处理函数中的this指向会丢失，其实真正调用时并不是this.handleClick(),如果是这样调用那么this指向就不会有问题。真正调用时是onClick()。

组件通讯

  • 组件传参props

    • 函数组件通过组件名 function(props)接收。在组件内使用方式 props.xxx
    • 类组件，组件内使用参数：this.props.xxx

  • props特点

    • 可以传递任意类型参数
    • props传递的参数都是只读属性
    • 在class类组件constructor函数中super中必须先将props作为参数传递进去。否则获取不到Props

  • 组件通讯

    • 父传子

    • 子传父

      • 原理，父组件定义回调函数，形参即为接收的数据。将回调函数作为数据通过props传递给子组件，由子组件在调用时将数据作为实参传递给父组件

// 父组件
class Parent extends React.Component {
  //定义state
  state = {
    lastName: '',
    count: 5,
  }
  GetChildData = () => {
    var Bcount = this.state.count
    console.log('Bcount=>', Bcount)
    // console.log('接收到子组件传递的数据', data)
    this.setState({
      count: (Bcount += 1),
    })
  }

  render() {
    return (
      <div>
        <span>父组件接收到子组件传递的值：{this.state.count}</span>
        <Tom count={this.state.count} GetChildData={this.GetChildData}></Tom>
        <Bob count={this.state.count} GetChildData={this.GetChildData}></Bob>
      </div>
    )
  }
}

/* 子组件1 */
class Tom extends React.Component {
  state = {
    sonName: 'xiaoZhao',
    // tcount:''
  }
  sendData = () => {
    this.props.GetChildData()
  }
  render() {
    return (
      <div>
        子组件Tom:
        <h2>count:{this.props.count}</h2>
        <button onClick={this.sendData}>点我向父组件传递数据</button>
      </div>
    )
  }
}
