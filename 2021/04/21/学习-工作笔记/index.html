前端自动化导入

  • 通过webpack提供的require.context()方法来实现自动化导入

  • 该方法接收三个参数require.context(directory,
    (useSubdirectories = true),
    (regExp = /^./.*$/),) 第一个参数为路径，第二个为Boolean值，表示是否查找子目录。第三个参数是正则表达式。可以自定义规则进行匹配。

  • require.context()返回一个函数webpackContext。该函数相当于require，不过其接收的模块路径只能是require.context查找到的模块。

  • webpackContext还有两个静态方法（resolve,keys）和一个属性(id);

    • resolve 接收参数为匹配成功的模块路径，即keys中存储的值；其返回匹配文件相对稳整个项目的相对路径。
    • keys:返回成功获取的模块路径组成的数组。
    • id:上下文模块的模块id。

  • 由于浏览器必须要在执行事件处理函数之后，才能知道有没有掉用过 preventDefault() ，

在 window、document 和 body 上注册的 touchstart 和 touchmove 事件处理函数，
会默认为是 passive: true。浏览器忽略 preventDefault() 就可以第一时间滚动了。

举例：
wnidow.addEventListener(‘touchmove’, func) 效果和下面一句一样
wnidow.addEventListener(‘touchmove’, func, { passive: true })

    • 为容器设置css属性touch-action，去除滑动时默认现象产生，但不影响事件触发

当el-ment中form表单项为v-if动态渲染时验证规则添加失败

  • 原因:牵扯到diff算法，复用问题。因为form-item绑定验证事件是在mounted中进行的，规则变化后没有进行重新绑定验证事件，v-if渲染组件节点diff后被复用了，所以验证也就自然失效了
  • 解决办法：给item添加key属性，保证item的唯一。这样就可以动态绑定规则了

vue中触发视图更新与不能触发视图更新的操作

  • 能够触发视图更新的操作（数组）

    • push
    • splice
    • unshift
    • pop
    • shift
    • sort
    • reverse

  • 不能够触发更新的操作(数组：这些方法基本都是返回一个新的数组，vue不能监测到)

    • concat
    • slice
    • filter
    • 直接改变数组的长度如arr.length=newlength
    • 直接改变数组的某项值如arr[index]=newValue

  • 对象，在初始化data赋值后给数组添加新的属性，也不能触发视图的更新

  • 解决方法

    • 利用vue中的api：this.$set(obj|arr,prop|index,newValue)来给对象或数组添加响应式的属性或值，但是这个方法不能给vue实例或者vue的根数据源添加修改属性

    • 对于对象可以利用Object.assgin()方法

      • this.obj=Object({},this.obj,{propa:"",propb:""})

使用XLSX导出el-table表格数据重复的问题

  • 问题描述：xlsx导出el-table表格数据时，出现数据重复导出问题，经查验，table表格数据无重复，导出方式也无问题。最后经查阅发现，如果给el-table表格中的列添加了fixed属性，在导出时会导致表格重复。

  • 解决方法：在导出前做校验，移除fixed属性，然后再导出就不会出现数据 重复了。

  • 代码：

    •  //  .table要导出的是哪一个表格,id：表格id
      let table = document
        .querySelector(id) 
        .cloneNode(true);
      // console.log("table", table);
      // 因为element-ui的表格的fixed属性导致多出一个table，会下载重复内容，这里删除掉
      table.removeChild(table.querySelector(".el-table__fixed"));
      let vb = XLSX.utils.table_to_book(table, { raw: false });

      let vbout = XLSX.write(vb, {
        bookType: "xlsx",
        bookSST: false,
        type: "array"
      });

      try {
        FileSaver.saveAs(
          new Blob([vbout], { type: "application/octet-stream" }),
          this.$root.stationType.NM +
            " 年特征值表(" +
            (this.date.length > 4
              ? this.formatDate(new Date(this.date), "yyyy年MM月")
              : this.formatDate(new Date(this.date), "yyyy年")) +
            ").xlsx"
        );
      } catch (e) {
        if (typeof console !== "undefined") console.log(e, vbout);
      }
      return vbout;

获取元素距离可视区域的距离：element.getBoundingClientRect()

  • 该方法返回一个rect对象，该对象是由该元素的getClientRect()方法返回的一组矩形集合，包含完整元素的最小矩形。rect是一个具有left、right、top、bottom的DOMRect对象。
  • left、right、top、bottom表示当前盒子距离可视区域的各个方向距离
  • 当计算边界矩形时，会考虑到当前视口区域的滚动操作，也就是说当滚动位置发生了改变，rect的对象值也会随之发生改变。如果你需要获得相对于当前页面左上角的的坐标值，只需给left、top加上window.scrollLeft、window.scrollTop（为了照顾浏览器的兼容性可以使用window.pageXoffset和window.pageYoffset替代）。就可以得到与当前滚动位置无关的值。
  • 获取可视区域的相关数据:window.innerHeight、window.innerWidth、document.documentElement.clientHeight、document.documentElement.clientWidth
  • 根据这些信息可以实现一些如让元素滚动到可视区域再加载动画或者图片的操作

ajax实现图片/文件上传遇到的问题

  • 问题描述：ajax上传文件，后端给定的上传格式为formData格式，但是当用ajax手动上传设置contentType: “multipart/form-data”时，会报错the request was rejected because no multipart。
  • 问题分析:
    • 当直接利用表单提交时，将表单的enctype设置为’multipart/form-data’可以实现。
    • 当用ajax发送请求时，则需要将配置项contentType设置为false，而不能直接设置为”multipart/form-data”，后者发送的请求在Request headers的contentType中没有分界符，会导致后台无法解析并报错。

前端携带url做为路径参数时请求报错的问题

  • 直接将url地址作为参数接在地址栏会出现报错问题
  • 解决方法：使用 encodeURIComponent(url)函数或者encodeURI(url)转换一下，将地址编码后再进行传递就可避免这个问题
