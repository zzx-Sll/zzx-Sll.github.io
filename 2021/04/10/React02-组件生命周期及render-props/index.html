React组件的生命周期

组件的生命周期：组件从创建到挂载到卸载的过程

创建时的生命周期

  • constructor:构造函数
    • 作用：设置state数据，this的指向设置
  • render：渲染函数
    • 作用：渲染页面。注意：不要在此函数中使用setState()
  • componentDidMount:组件挂载到页面后触发的函数
    • 作用：1 发送网络请求 2：获取dom结构
  • 执行的顺序：constructor–>render–>componentDidMount

组件更新时的生命周期

  • render():每次组件渲染都会触发
    • 作用：渲染ui
  • componentDidUpdate():组件更新后触发
    • 作用：1发送网络请求 2DOM操作 3：如果使用setState()必须放在一个if条件中，发送网络请求同理
    • 注意：if条件可以是判断props的不同，上次的props：prevProps,此次的props：this.props

组件卸载时的生命周期

  • componentWillUnmount:组件卸载后触发
    • 作用：执行清理工作（比如：清理定时器等）

组件的封装及复用

React组件的复用概述
  • 如果两个组件中的部分功能相似或相同，该如何处理
  • 处理方式：复用组件，封装
  • 具体复用的是哪部分？
    • state
    • 操作state的方法
  • 两种方式:
    • render props模式
    • 高阶组件HOC
  • 注意：这两种方式不是新的API，而是利用React自身的特点，演化而成的固定模式
    render-props思路分析
  • 思路：将要复用的state和操作state的方法封装到一个组件中
  • 如何拿到该组件中的state?
    • 在使用该组件时，添加一个值为函数的prop，通过函数参数来获取。
  • 如何渲染到任意UI？
    • 使用该函数的返回值作为要渲染的UI内容
      使用步骤
  • 创建Mouse组件，在组件中提供复用的逻辑代码
  • 将要复用的状态作为props.render函数参数，暴露到组件外部
  • 使用props.render()的返回值作为渲染的内容
代码优化（添加props验证，采用children属性代替render函数）

高阶组件封装可复用组件

概述
  • 目的：实现状态逻辑复用
  • 采用：包装模式
  • 手机：获取保护功能
  • 手机壳：提供保护功能
  • 高阶组件相当于手机壳，通过包装功能，给组件增强功能
    思路分析
  • 高阶组件 HOC 是一个函数，接收要包装的组件，返回增强后的组件
  • 高阶组件中创建了一个组件，提供state状态和操作状态的方法，将被包装的组件挂载到该组件的render()函数，然后通过props将复用的状态传递给被包装的组件。
    使用步骤
  • 创建一个函数，名称约定以with开头
  • 指定函数参数为被包装的组件
  • 创建一个组件，实现复用的逻辑代码，组件内部挂载被封装的函数，props传递复用的状态
  • return 包装后的组件
  • 调用高阶组件函数，同时将返回后的组件挂载到dom中
    设置displayName
  • 使用高阶组件存在的问题：得到两个组件的名称相同
  • 原因：默认情况下，React使用组件名称作为displayName
  • 解决方式：为高阶组件设置displayName，便于调试时区分不同的组件
  • 设置方式：
    • 传递props
  • 问题：如果没有传递props，会导致props丢失问题
  • 解决方式：挂载被封装的组件时，将props也一起传递给组件
